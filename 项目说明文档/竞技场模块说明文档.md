# 王国竞技场模块说明文档

> 本文档与当前代码保持一致，用于后续维护参考。  
> 对应代码路径：`帅斌饼干/脚本/game/常规_王国竞技场/`

---

## 1. 模块概述

王国竞技场模块自动化完成《冲呀！饼干人：王国》中的「王国竞技场」日常对战。

当前实现采用**过程式循环**，主流程为：

```
进入竞技场大厅 → 读取门票/奖牌/奖杯 → 循环：
  ├─ 检查战斗上限
  ├─ 门票不足时尝试自动买票
  ├─ 扫描当前页 → 翻页扫描 → 开战 / 刷新 / 退出
  └─ 单场战斗结束后回到读取状态
```

能力：

- 自动从王国首页进入竞技场大厅。
- 动态扫描当前页对手，按奖杯差与「已战斗」状态过滤。
- 当前页无合适敌人时左滑翻页再扫描一次。
- 两页均无合适敌人时检测免费刷新；可刷新则刷新并继续，不可刷新则解析倒计时后退出。
- 门票不足时按配置自动买票 1 次。
- 自动战斗、OCR 读取胜负结果、统计胜率。
- 安全离开竞技场返回王国首页。

## 3. 模块职责

### 3.1 竞技场_特征库.lua

集中定义竞技场所有图像特征、OCR 区域与点击坐标。UI 改版或分辨率适配时优先修改此文件。

| 分区 | 用途 |
|---|---|
| `lobby` | 大厅特征、关闭按钮、奖牌/门票 OCR、奖杯 OCR、刷新按钮 OCR、买票按钮与滑条 |
| `opponent` | 对手查找特征、战力/奖杯 OCR 区域、已战结果颜色识别 |
| `teamSelect` | 队伍选择页特征、开始战斗按钮 |
| `dialog` | 「部署更多饼干」「未装载配料」弹窗特征与确认按钮 |
| `battle` | 战斗开始 UI 特征 |
| `settlement` | 结算页特征、结果 OCR、离开按钮、升段跳过坐标 |
| `pagination` | 左右滑翻页坐标 |

### 3.2 竞技场_页面.lua

封装竞技场 UI 交互，向上层提供语义化 API。

| 函数 | 说明 |
|---|---|
| `isLobby()` / `waitLobby(timeoutMs)` | 大厅检测 / 等待 |
| `ensureLobby()` | 兜底恢复：若在大厅返回 true；若在结算页则离开结算页 |
| `tapClose(delayMs)` | 点击大厅关闭按钮 |
| `readMedalAndTicket()` | OCR 读取奖牌与门票数量 |
| `readTrophyCount()` | OCR 读取当前奖杯数 |
| `readOpponentInfo(location)` | 读取指定位置对手的坐标、战力、奖杯、是否已战、结果 |
| `findFirstValidOpponent(cfg, myTrophy)` | 扫描当前页，返回第一个符合要求的对手 |
| `swipePageLeft()` / `swipePageRight()` | 对手列表左滑 / 右滑 |
| `runBattle()` | 完整单场战斗流程 |
| `isFreeRefresh()` / `tapFreeRefresh()` | 检测 / 点击免费刷新 |
| `readRefreshCountdown()` | 解析刷新倒计时文本为秒数 |
| `waitFreeRefresh()` | 阻塞等待免费刷新出现（当前未使用） |
| `buyTicket()` | 打开买票弹窗、拖动滑条、点击确认 |

#### 3.2.1 整页对手扫描 `findFirstValidOpponent(cfg, myTrophy)`

```
findMultiColorAllT 获取当前页所有对手位
按位置 1..N 遍历：
  readOpponentInfo(loc)
  若 isBattled：跳过
  若我方奖杯 > 对手奖杯 且 差值 >= trophyDiff：跳过（奖杯过滤）
  否则：返回该对手信息
无合适对手返回 nil
```

#### 3.2.2 单场战斗流程 `runBattle()`

```
等待队伍选择页（30s）
  → 点击开始战斗
  → 若弹出「部署更多饼干」则确认
  → 若弹出「未装载配料」则确认
  → 等待队伍选择页消失（30s）
  → 等待结算页（120s）
  → OCR 读取结果
  → leaveSettlement()
```

> 说明：原实现通过识别「战斗开始 UI」判断是否进入战斗，因识别不准改为通过 `Color.waitGone` 等待队伍选择页消失来判断。

返回战斗结果字符串（`胜利`/`平局`/`失败`）或 `nil`。

#### 3.2.3 离开结算页 `leaveSettlement()`

```
已在大厅 → 返回 true
不在结算页且无离开按钮 → 返回 false（交给调用方处理）
循环点击 leaveRetryTap 直到出现离开按钮（最多 60 次，间隔 1200ms）
  → 点击 leaveTap，等待 2s
  → 循环点击 promotionSkip 直到出现大厅特征（最多 60 次，间隔 1000ms）
  → 等待 2s 后若仍在大厅返回 true；否则再次循环点击 promotionSkip
```

### 3.3 竞技场_路由.lua

| 函数 | 流程 |
|---|---|
| `enter()` | 已在大厅则跳过；否则主城 → 冒险页 → OCR 点击「王国竞技场」→ 等待大厅（每步 30s 超时） |
| `leave()` | 已回主城则跳过；大厅则点击关闭，冒险页则按返回键，等待回到主城（15s 超时） |

### 3.4 竞技场_任务.lua（过程式循环版）

不再使用 `core.state-machine`，主流程由 `while` 循环驱动。

内部函数：

| 函数 | 职责 |
|---|---|
| `syncSession()` | 读取奖牌/门票/奖杯，更新会话（不刷新 HUD） |
| `syncStatus(cfg)` | 调用 `syncSession()` 并刷新 HUD 为完整战斗统计 |
| `isReachMaxBattles(cfg)` | 判断是否达到 `maxBattles` |
| `tryAutoBuy(cfg)` | 无票时尝试买 1 次票，买到票返回 true |
| `doBattle(info, cfg)` | 点击对手、执行单场战斗、更新胜负统计与门票 |
| `selectAndFight(cfg)` | 扫描 → 翻页扫描 → 刷新/退出，返回动作标识 |

#### 3.4.1 主循环 `Task.run()`

```lua
function Task.run()
    -- 1. 前置检查
    if cfg.enabled ~= true then return false end
    if isReachMaxBattles(cfg) then return false end

    -- 2. 进入竞技场
    if not ArenaRoute.enter() then return false end

    -- 3. 业务循环
    while running do
        syncStatus(cfg)

        -- 3.1 上限检查
        if isReachMaxBattles(cfg) then break end

        -- 3.2 门票检查与自动买票
        if tickets <= 0 then
            if not tryAutoBuy(cfg) then break end
            syncStatus(cfg)
        end

        -- 3.3 对手选择与战斗
        local result = selectAndFight(cfg)
        if result == false then
            -- 战斗失败，终止
            running = false
        elseif result == "exit" then
            running = false
        elseif result == "refreshed" then
            -- 继续循环，从第 1 页重新扫描
        end
        -- "fought" 继续循环
    end

    -- 4. 离开
    syncSession()
    ArenaRoute.leave()
    return true
end
```

#### 3.4.2 对手选择 `selectAndFight(cfg)`

```
扫描当前页 findFirstValidOpponent
  → 有：开战，返回 "fought"
  → 无：swipePageLeft 翻页
    → 再扫描
      → 有：开战，返回 "fought"
      → 无：检查 isFreeRefresh
        → 是：点击刷新、右滑复位、返回 "refreshed"
        → 否：解析倒计时持久化、返回 "exit"
```

#### 3.4.3 对手过滤规则

沿用现有规则：

- 跳过 `isBattled == true` 的对手。
- 若我方奖杯 > 对手奖杯且差值 ≥ `cfg.trophyDiff`，跳过。
- 其余情况视为可开战。

含义：只打接近或高于我方的对手，避免虐低奖杯对手。

#### 3.4.4 自动买票 `tryAutoBuy(cfg)`

```
若 autoBuyCount <= 0：离开
若 buyCount >= autoBuyCount：离开
否则：
  buyTicket()
  buyCount + 1
  等待 1500ms
  syncStatus()
  若门票 > 0：返回 true（继续）
  否则：离开
```

> 注意：`buyTicket()` 未校验购买结果，依赖后续 OCR 读取门票数判断。

#### 3.4.5 免费刷新与倒计时

```
若 isFreeRefresh() 为 true：
  点击免费刷新
  清除 nextFreeRefreshAt
  swipePageRight 复位到第 1 页
  返回 "refreshed"
否则：
  readRefreshCountdown()
  若解析成功：nextFreeRefreshAt = now + seconds
  否则：默认 30 分钟后重试
  返回 "exit"
```

### 3.5 竞技场_会话.lua

持久化键：`arena_session`。

| 字段 | 说明 |
|---|---|
| `wins` | 胜利场次 |
| `losses` | 失败场次 |
| `draws` | 平局场次 |
| `tickets` | 当前门票数 |
| `buyCount` | 已自动买票次数 |
| `trophies` | 当前奖杯数 |
| `medals` | 奖牌数 |
| `nextFreeRefreshAt` | 下次免费刷新时间戳（秒） |

API：

- `get()` / `set(data)` / `update(partial)`
- `totalBattles()`：胜 + 负 + 平
- `describe()`：格式化统计字符串
- `clear()`：清空会话
- `setNextFreeRefreshAt(at)` / `getTimeUntilRefresh()` / `clearNextFreeRefresh()`

## 4. 用户配置

配置入口：`帅斌饼干/脚本/config.lua` 的 `STATIC.USER.arena`。

```lua
arena = {
    enabled = false,      -- 功能开关
    maxBattles = nil,     -- 战斗上限，nil 表示无限
    autoBuyCount = 0,     -- 自动买票次数，0 表示不买
    trophyDiff = 0,       -- 奖杯差值阈值
}
```

UI 配置位置：

- 功能开关：`ui/tabs/feature-tab.lua` → 「王国竞技场」复选框
- 参数配置：`ui/tabs/config-tab.lua` → 「王国竞技场」分组
  - 战斗上限（空 = 无限）
  - 自动买票次数
  - 奖杯差值阈值

## 5. 调度注册

`game/register.lua` 中注册为调度任务：

```lua
Scheduler.add("王国竞技场", conditionFn, taskFn)
```

**condition 条件：**

1. `arena.enabled == true`
2. 矿山相关调度空闲（矿山优先）
3. 未达到 `maxBattles` 战斗上限
4. 免费刷新倒计时已到期

**action 执行：**

1. 若处于广场上下文，先调用 `SquareTask.leaveForOtherTask()` 离开广场。
2. 调用 `ArenaTask.run()`。

## 6. 关键数据流与边界

### 6.1 门票扣除

- 战斗结束后由任务模块手动 `tickets - 1`。
- 若 `runBattle()` 实际未消耗门票，会话门票数可能偏离真实值。

### 6.2 刷新等待

- 调度侧：倒计时未到期时跳过任务，HUD 显示 `刷新等待 Xs`。
- 运行侧：`core/runtime.lua` 已将竞技场刷新倒计时纳入空闲等待计算，到期前会唤醒执行。

### 6.3 战斗超时

- 队伍选择页等待：30s
- 队伍选择页消失等待（进入战斗标志）：30s
- 结算页等待：120s

### 6.4 翻页与复位

- 对手选择最多翻页一次（当前页 → 翻页后）。
- 点击免费刷新后，调用 `swipePageRight()` 确保列表回到第 1 页，避免下一轮扫描错误页。

## 7. 注意事项与维护提示

1. **UI 适配**：界面变化时优先检查 `竞技场_特征库.lua`。
2. **OCR 识别**：对手战力/奖杯使用专用 `numberOcr` 配置，识别失败通常与颜色、字号或背景变化有关。
3. **买票结果未校验**：当前只点击操作，成功性依赖后续 OCR 读取。
4. **无升段判定**：当前流程图已移除升段判定，升段后列表不会自动重置。
5. **会话清理**：用户可通过 UI 「重置」按钮清除 `arena_session`，重新开始统计。
