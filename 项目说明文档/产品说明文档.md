网页远程操控模拟器(懒人精灵端)实施计划
核心结论(基于本地文档核实,推翻前期错误判断)
懒人精灵原生支持 WebSocket 客户端(startWebSocket/sendWebSocket/closeWebSocket,见网络方法文档 399-457 行),且支持多线程(beginThread,最多10线程)、OpenCV 扩展(cv.snapShot/cv.resize/cv.imwrite)、root 环境下全套触控(含 touchDown/Move/Up 多点)。

因此架构从最初设想的"HTTP 轮询"改为 WebSocket 全链路实时方案——延迟最低、实现最简洁。

通信协议(三端统一)
所有消息均为 JSON 文本帧,带 type 字段区分。

设备 → 服务器:

type	字段	说明
register	id, brand, model, width, height	上线注册(复用 device-id.lua 的设备码)
frame	img(base64 JPEG), w, h, ts	画面帧(线程A推送)
ack	cmdId	指令执行回执(可选)
浏览器 → 服务器:

type	字段	说明
subscribe	ids[]	订阅设备画面
command	target(设备id或"broadcast"), action, ...	单控或群控指令
服务器 → 设备:

type	字段	说明
cmd	seq, action, x, y, ...	指令(action见下)
服务器 → 浏览器:

type	字段	说明
frame	id, img, w, h	转发设备画面
devices	list[]	设备列表变更(上线/下线)
action 指令集(root 环境全套可用):

tap(x,y) / longpress(x,y,dur) — 用 tap/longTap
swipe(x1,y1,x2,y2,time) — 用 swipe
touch(phase:down/move/up, id, x, y, time?) — 用 touchDown/touchMove/touchMoveEx/touchUp(多点/精细手势)
key(name) — 用 keyPress(home/back/recent/power)
input(text) — 用 inputText

-- encoder.lua 核心逻辑(伪代码)
function encodeFrame(w, h)
    local mat = cv.snapShot(0,0,w,h)              -- 截图返Mat(纯内存)
    if w > SCALE_MAX_WIDTH then
        local scaled = cv.Mat.new()
        cv.resize(mat, scaled, ...)                -- 降采样
        mat:release(); mat = scaled
    end
    -- 验证步骤:先测 cv.imencode 是否可用
    if cv.imencode then
        local buf = cv.imencode(".jpg", mat, {cv.IMWRITE_JPEG_QUALITY, JPEG_QUALITY})
        b64 = encodeBase64(buf)                    -- 纯内存
    else
        -- 降级:写临时JPEG文件再读base64(仍是JPEG,省带宽)
        cv.imwrite(TMP_JPG, mat)
        b64 = getFileBase64(TMP_JPG)
        delfile(TMP_JPG)
    end
    mat:release()
    return b64
end

server.ts 骨架:


ts
Bun.serve({
    port: 8080,
    fetch(req, server) {
        const url = new URL(req.url);
        if (url.pathname === "/ws" && server.upgrade(req)) return; // WS升级
        return new Response(Bun.file("./web/index.html"));          // 否则返回前端
    },
    websocket: {
        open(ws) { /* 等首条消息区分 role */ },
        message(ws, msg) { /* JSON解析→router分发 */ },
        close(ws) { /* 清理注册表 */ },
    }
});
群控核心逻辑(router.ts):


ts
function handleCommand(msg) {
    const targets = msg.target === "broadcast"
        ? [...registry.keys()]              // 群控:所有在线设备
        : [msg.target];                     // 单控:指定id
    for (const id of targets) {
        const dev = registry.get(id);
        dev?.ws.send(JSON.stringify({type:"cmd", seq:nextSeq(), ...msg}));
    }
}

关键技术风险与缓解
cv.imencode 是否暴露到 Lua 绑定(文档未明确):实施第一步先跑诊断代码确认。缓解:已内置降级路径——cv.imwrite 临时 JPEG 文件 + getFileBase64,仍是 JPEG(比 PNG 省 ~33% 带宽),仅多一次磁盘 I/O。
WebSocket 回调线程限制(文档明确:回调在独立线程,不能调 exitScript/restartScript):重连用 setTimer 切回主线程;触控 API 在回调线程调用是安全的。
大图带宽:默认降采样到 720 宽 + JPEG 质量 60,单帧约 30-50KB,10fps 约 300-500KB/s/设备,两台可接受。config.lua 可调。
