# 矿山开采流程

> 对应代码：`帅斌饼干/脚本/game/常规_未知的地底矿山/模块_矿山开采/`  
> 最后更新：2026-06-15

---

## 一、宏观：从调度到再次执行

```mermaid
flowchart TD
    subgraph scheduler [调度层 register.lua]
        A[调度器每轮检查] --> B{miningEnabled?}
        B -->|否| SKIP[跳过开采]
        B -->|是| C{MiningSession.checkReady<br/>busy 已到期?}
        C -->|否| WAIT[跳过 · HUD 显示开采剩余时间]
        C -->|是| D{在广场?}
        D -->|是| E[离开广场]
        D -->|否| F[MiningTask.run]
        E --> F
    end

    subgraph task [任务层 开采_任务.lua]
        F --> G[状态机 detect → … → recordDone]
        G --> H[Route.returnToKingdom 回王国]
        H --> I[MiningSession.enterBusyWait<br/>默认 6 小时]
        I --> DONE[DONE 本轮结束]
    end

    DONE --> A
    WAIT --> A
    SKIP --> A
```

**要点：**

- 与勘查任务**独立调度**，互不阻塞
- 本轮结束后写入 `busy` 截止时间（默认 6h），到期前调度器不再拉起
- 空闲时 HUD 可与勘查合并显示：`勘查远距 Xs · 开采 Ys`

## 二、状态机总览（核心）

```mermaid
flowchart TD
    START([MiningTask.run 启动]) --> detect

    detect{detect<br/>识别当前页面}
    detect -->|开采页| miningPage
    detect -->|矿山首页| precheck
    detect -->|王国首页| navigate
    detect -->|都不匹配| RETRY1[RETRY / 失败]

    navigate{navigate<br/>导航}
    navigate -->|王国首页| N1[活动 → 矿山首页]
    navigate -->|矿山首页| precheck
    navigate -->|开采页| miningPage
    N1 --> KEEP1[KEEP 等待下轮]

    precheck{precheck<br/>首页预检}
    precheck -->|点开采进入| miningPage
    precheck -->|失败| RETRY2[RETRY]

    miningPage{miningPage<br/>开采页调度}
    miningPage -->|结算/奖励页| claimConfirm
    miningPage -->|有已完成矿卡| claimTap
    miningPage -->|有空闲栏位| selectFlow
    miningPage -->|有可开始矿卡| startFlow
    miningPage -->|以上都没有| checkSlot

    claimTap[claimTap<br/>点击已完成矿卡] --> claimConfirm
    claimConfirm[claimConfirm<br/>确认奖励回到开采页] --> miningPage

    checkSlot{checkSlot<br/>复查栏位}
    checkSlot -->|有空闲| selectFlow
    checkSlot -->|可开始| startFlow
    checkSlot -->|都没有| recordDone

    selectFlow[selectFlow<br/>选卡填栏] --> miningPage
    startFlow[startFlow<br/>启动矿卡] -->|仍有可开始| startFlow
    startFlow -->|全部启动完| miningPage
    startFlow -->|无可开始| recordDone

    recordDone{recordDone<br/>结束复查}
    recordDone -->|发现空闲| selectFlow
    recordDone -->|发现可开始| startFlow
    recordDone -->|发现已完成| claimTap
    recordDone -->|确认无可操作| END([busy 6h + 回王国 + DONE])
```

**状态机参数：** `maxRetry=3`，`timeout=1800s`，轮询间隔 `500ms`。

## 三、`miningPage` 决策优先级

开采页每次进入按**固定顺序**判断，先匹配先执行：

| 优先级 | 条件 | 下一状态 | 说明 |
|--------|------|----------|------|
| 1 | 结算路径 / 奖励页 | `claimConfirm` | OCR「获得开采奖励」或首页已离开且非开采页 |
| 2 | 有已完成任务 | `claimTap` | 绿色完成标记矿卡 |
| 3 | 有空闲栏位 | `selectFlow` | 除非 `skipSelectOnce`（上轮选卡全失败） |
| 4 | 有可开始矿卡 | `startFlow` | 已选卡、待点「开始」 |
| 5 | 以上都没有 | `checkSlot` | 二次确认后可能结束 |

## 四、`selectFlow` 选卡子流程

```mermaid
flowchart TD
    S0[点空闲栏位] --> S1[进入「选择多个」多选模式]
    S1 --> S2[OCR 读取配额 cur/max]
    S2 --> S3{按优先级逐类选卡<br/>amberFossil → sugarOre → …}

    S3 --> S4[图色找目标矿卡]
    S4 --> S5[点击选中 · OCR 校验 +1]
    S5 --> S6{本屏还有目标卡?}
    S6 -->|有进展| S4
    S6 -->|无进展| S7[左滑卡列表 · 最多 20 次]
    S7 --> S8{该类矿卡耗尽?}
    S8 -->|是| S9[尝试下一优先级]
    S8 -->|否| S3
    S9 --> S3

    S3 -->|cur=0 全选不了| S10[返回开采页 · skipSelectOnce=true]
    S10 --> miningPage

    S3 -->|选够或尽力| S11[确认选卡]
    S11 --> S12[等待回到开采页]
    S12 --> miningPage
```

**选卡优先级来源：** `UserConfig.mine.miningOreCards`（UI 勾选顺序），默认：

1. 琥珀化石（amberFossil）
2. 糖矿石（sugarOre）
3. 紫化石（purpleFossil）
4. 绿化石（emeraldFossil）
5. 面粉石（flourStone）

## 五、`startFlow` 启动子流程

```mermaid
flowchart TD
    T0{有可开始矿卡?} -->|否| recordDone
    T0 -->|是| T1[点击可开始矿卡]
    T1 --> T2[等待 setup 准备页]
    T2 --> T3[自动选饼干]
    T3 --> T4[点开始开采]
    T4 --> T5[处理确认弹窗<br/>勾选今日不再显示]
    T5 --> T6[等待回到开采页]
    T6 --> T7{仍有可开始矿卡?}
    T7 -->|是| KEEP[KEEP 同状态下一轮]
    KEEP --> T0
    T7 -->|否| miningPage
```

## 八、配置项

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `miningEnabled` | `false` | 是否开启开采 |
| `miningIntervalSec` | `21600`（6h） | 本轮结束后再次调度间隔 |
| `miningOreCards` | 见上文优先级 | 选卡种类与顺序 |

## 九、一次完整运行示例

```
调度到期 → 离开广场（如在广场）
  → detect（王国）→ navigate → precheck
  → miningPage
      ├─ 有空位 → selectFlow（按优先级选满）→ miningPage
      ├─ 有可开始 → startFlow（可能多轮 KEEP）→ miningPage
      ├─ 有已完成 → claimTap → claimConfirm → miningPage（循环领奖）
      └─ 栏位全满且都在跑 → checkSlot → recordDone
  → enterBusyWait(6h) → returnToKingdom → DONE
  → 调度器等待 6 小时后再次执行
```
