# 项目任务调度流程图

> 核心模块：`core/runtime.lua`、`core/scheduler.lua`、`core/guard.lua`、`game/register.lua`

---

## 1. 启动链路

```mermaid
flowchart TD
    A[启动 APP] --> B[HotUpdate.runOnStartup]
    B --> C[License.init]
    C --> D[Ocr.init]
    D --> E[创建 imgui 窗口]
    E --> F[用户点击 启动脚本]
    F --> G[License.verify]
    G -->|失败| H[toast 提示]
    G -->|成功| I[关闭窗口]
    I --> J[Runtime.register = Register.all]
    J --> K[Runtime.run]
```

---

## 2. 运行时主循环

```mermaid
flowchart TD
    A[Runtime.run] --> B[Scheduler.clear]
    B --> C[Guard.clear]
    C --> D[StatusHud.init]
    D --> E[Runtime.register 注入业务]
    E --> F[Color.setGuardHook Guard.check]
    F --> G[while true 主循环]
    G --> H[round++]
    H --> I[Guard.check 扫描弹窗]
    I --> J[Scheduler.run]
    J -->|hasWork=true| K[Guard.sleep stepMs 5s]
    J -->|hasWork=false| L[计算最近等待时间]
    L --> M{有等待任务?}
    M -->|是| N[每秒刷新 HUD 等待倒计时]
    M -->|否| O[Guard.sleep idleMs 30s]
    N --> P[等待到期或 30s]
    P --> G
    O --> G
    K --> G
```

---

## 3. 调度器单轮执行

```mermaid
flowchart TD
    A[Scheduler.run] --> B[遍历已注册任务]
    B --> C{任务 condition?}
    C -->|异常| D[记录 warn]
    C -->|false| E[加入 skipped 列表]
    C -->|true| F[StatusHud 显示执行中]
    F --> G[pcall task.action]
    G -->|异常| H[记录 error]
    H --> I{stopOnError?}
    I -->|是| J[返回 hasWork=false]
    I -->|否| K[继续下一个任务]
    G -->|成功| L[记录耗时]
    L --> K
    E --> K
    D --> K
    K --> M{还有任务?}
    M -->|是| B
    M -->|否| N[返回 hasWork]
```

---

## 4. 任务优先级与条件判断

```mermaid
flowchart LR
    subgraph 注册顺序
        A1[矿山勘查] --> A2[矿山开采]
        A2 --> A3[海滩交易所]
        A3 --> A4[王国竞技场]
        A4 --> A5[布谷鸟广场]
        A5 --> A6[洗脆饼词条]
    end

    subgraph 矿山优先策略
        B1{isMineSchedulerIdle?}
        B1 -->|否| B2[非矿山任务跳过]
        B1 -->|是| B3[检查自身条件]
    end

    A1 -.->|无条件最高优| B1
    A2 -.->|无条件最高优| B1
    A3 -.->|受矿山空闲约束| B1
    A4 -.->|受矿山空闲约束| B1
    A5 -.->|受矿山空闲约束| B1
    A6 -.->|不受矿山约束| B3
```

---

## 5. 守卫（Guard）扫描

```mermaid
flowchart TD
    A[Guard.check] --> B[按 priority 降序排序 traps]
    B --> C[遍历 traps]
    C --> D{Color.any feature?}
    D -->|是| E[pcall handler]
    E -->|异常| F[记录 error 返回 false]
    E -->|成功| G[记录 info 返回 true]
    D -->|否| H[下一个 trap]
    H --> C
    C --> I[无命中 返回 false]
```

---

## 6. 空闲等待策略

```mermaid
flowchart TD
    A[无任务执行] --> B[读取四个等待时间]
    B --> C[farRemain 矿山勘查远距]
    B --> D[miningRemain 矿山开采 busy]
    B --> E[marketRemain 海滩交易所补货]
    B --> X[arenaRemain 竞技场刷新]
    C --> F[max = 最大值]
    D --> F
    E --> F
    X --> F
    F --> G{max > 0?}
    G -->|是| H[循环每秒检查]
    G -->|否| I[显示空闲 睡 30s]
    H --> J{到期?}
    J -->|否| K[刷新 HUD]
    K --> L[Guard.sleep 1s]
    L --> H
    J -->|是| M[退出等待]
```

---

## 7. 扩展新任务位置

```mermaid
flowchart TD
    A[新增业务模块] --> B[实现 condition + action]
    B --> C[在 game/register.lua 中 Scheduler.add]
    C --> D[决定优先级位置]
    D --> E{是否需要让矿山?}
    E -->|是| F[调用 isMineSchedulerIdle]
    E -->|否| G[直接判断自身开关]
    F --> H[任务接入调度]
    G --> H
```
